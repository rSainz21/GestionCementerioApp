# Devlog — 2026-04-16 — Gestión Cementerio Somahoz

Esta sección resume **lo trabajado el 16-04** para arrancar el módulo de **Gestión del Cementerio Municipal (Somahoz)** siguiendo el stack y convenciones de Conect@ 2.0.

## Contexto y objetivo

- **Objetivo**: arrancar desde cero el módulo de Cementerio (local), dejando listo:
  - esquema de BD (migraciones) **agnóstico SQLite/MySQL**
  - modelos Eloquent con relaciones
  - API mínima para el wizard “Nuevo Caso”
  - componente Vue crítico: **Selector de nichos en cuadrícula** (no `<select>`)
  - seed de datos realistas para ver libres/ocupados en el grid
  - scripts de instalación para Windows

- **Stack objetivo** (según `CLAUDE.md`):
  - Backend: Laravel 12.x + PHP 8.2
  - Auth: Sanctum (Bearer tokens)
  - Permisos: Spatie
  - Frontend: Vue 3 (Composition API, `<script setup>`) + Vite
  - UI target del ecosistema: PrimeVue 4 + preset Aura (pendiente de integrar visualmente de forma completa)

## Estado del repo hoy (qué hay implementado)

### 1) Base de datos (migraciones)

Se crearon migraciones para replicar el **schema de referencia** del SQL de producción (`basedatosnueva.sql`) de forma compatible con SQLite y MySQL.

Migraciones creadas en `database/migrations/`:
- `cemn_cementerios`
- `cemn_zonas` (FK a cementerio)
- `cemn_bloques` (FK a zona; `filas`/`columnas` + `tipo`)
- `cemn_sepulturas` (celdas; FK a zona y opcional a bloque; `fila`/`columna`/`estado`)
- `cemn_terceros` (personas)
- `cemn_concesiones` (FK a sepultura + auto-referencia `concesion_previa_id`)
- `cemn_concesion_terceros` (pivot concesión-tercero con `rol`)
- `cemn_difuntos` (FK opcional a tercero y a sepultura)
- `cemn_documentos` (FK a sepultura)
- `cemn_fuentes`
- `cemn_movimientos` (FK a difunto y sepulturas origen/destino)
- `cemn_registro_fuentes` (FK a fuente)

Notas importantes:
- En el SQL real la “unidad” individual es `cemn_sepulturas` (no `cemn_unidades`).
- Se evitó usar tipos exclusivos de SQLite (objetivo: migraciones válidas también para MySQL).

### 2) Modelos Eloquent

Modelos creados en `app/Models/` (todos con `$table` apuntando a `cemn_*`, `$fillable` y relaciones):
- `CemnCementerio` → `hasMany` zonas
- `CemnZona` → `belongsTo` cementerio; `hasMany` bloques y sepulturas
- `CemnBloque` → `belongsTo` zona; `hasMany` sepulturas
- `CemnSepultura` → `belongsTo` zona y bloque; `hasMany` concesiones/difuntos/documentos; helpers de estado + `generarCodigo()`
- `CemnTercero` → `belongsToMany` concesiones vía pivot; accessor `nombre_completo`
- `CemnConcesion` → `belongsTo` sepultura; `belongsToMany` terceros vía pivot; relación concesionario activo
- `CemnConcesionTercero` (pivot como modelo)
- `CemnDifunto` → `belongsTo` sepultura; `hasMany` movimientos
- `CemnDocumento`
- `CemnFuente`
- `CemnMovimiento`
- `CemnRegistroFuente`

### 3) API (mínima para wizard + grid)

Rutas en `routes/api.php`:
- `GET /api/cementerio/catalogo`
  - Controlador: `app/Http/Controllers/Cementerio/CementerioCatalogoController.php`
  - Devuelve `zonas` y `bloques` para alimentar selectores del frontend.
- `GET /api/cementerio/bloques/{bloque}/sepulturas`
  - Controlador: `app/Http/Controllers/Cementerio/BloqueSepulturasController.php`
  - Devuelve sepulturas del bloque con `difuntoTitular` precargado para tooltip.
- `POST /api/cementerio/nuevo-caso`
  - Controlador: `app/Http/Controllers/Cementerio/NuevoCasoController.php`
  - Guarda **transaccionalmente** (DB::transaction):
    - crea titular si no existe (o usa uno existente)
    - crea concesión
    - crea pivot `cemn_concesion_terceros` (rol `concesionario`)
    - crea difunto
    - **bloquea la unidad** (`lockForUpdate`) y cambia estado a `ocupada`

### 4) Frontend (Vue)

#### Servicio API (Axios)
- `resources/js/services/api.js`
  - incluye interceptor Bearer token `conecta2_token`
  - en 401 limpia token

#### Componente crítico: selector visual de nichos (Grid CSS)

  - selector de zona + bloque

  - render de celdas con colores:

    - rojo = ocupada (tooltip con nombre del difunto; no clic)

  - carga sepulturas desde `GET /api/cementerio/bloques/{id}/sepulturas`
  - emite `update:selectedSepulturaId` y `selected`

#### Wizard “Nuevo Caso” (esqueleto funcional)
- `resources/js/pages/cementerio/NuevoCasoWizard.vue`
  - 4 pasos: titular → difunto → unidad (grid) → resumen
  - llama a:

    - `POST /api/cementerio/nuevo-caso`
  - Nota histórica: el buscador en tiempo real del titular estaba pendiente aquí; **quedó resuelto el 17-04** (API `terceros?q=` + UI en el wizard).

#### SPA mínima para poder ver el wizard

### 5) Seed de datos (Somahoz)

Se creó:
- `database/database.sqlite` (archivo vacío listo para migrar)
- `database/seeders/CementerioSomahozSeeder.php`
  - crea 1 cementerio + 2 zonas
  - crea 3 bloques con cuadrículas:
    - Nichos A: 8×10
    - Nichos B: 6×12
    - Columbarios A: 5×8
  - genera todas las celdas (`cemn_sepulturas`) con códigos tipo `NA-F2-C3`
  - ocupa aleatoriamente algunas unidades creando:
    - tercero titular
    - concesión vigente
    - pivot concesionario

    - marca sepultura como `ocupada`
  - marca una `reservada` y una `clausurada` para ver estados extra
- `database/seeders/DatabaseSeeder.php` llama al seeder de Somahoz.

## Scripts de instalación en Windows

Por limitaciones del entorno (en Cursor/terminal no había PHP/Composer disponibles), se prepararon scripts para automatizar:

- `scripts/install-prereqs.ps1`
  - usa `winget` para instalar:
    - Laragon
    - Composer
    - Node.js LTS
  - ahora muestra salida y acepta acuerdos automáticamente (se quitó `Out-Null` y se añadieron flags de aceptación).

- `scripts/full-setup.ps1`
  - hace setup completo del proyecto:
    - detecta `php.exe` en Laragon aunque no esté en PATH
    - si no hay composer global, descarga `tools/composer.phar` y lo usa
    - si no existe `artisan`, genera un skeleton Laravel 12 en temporal y restaura el módulo encima
    - migra y seedea
    - hace `npm install` (y reintenta con `--legacy-peer-deps` si hiciera falta)

## Problemas reales encontrados hoy (y cómo se resolvieron)

1) **PowerShell ExecutionPolicy** bloqueaba scripts (`PSSecurityException`).
   - Solución: ejecutar con:
     - `powershell -ExecutionPolicy Bypass -File .\scripts\...ps1`

2) Los scripts se estaban ejecutando desde `C:\Windows\system32`.
   - Solución: `cd` al directorio del repo antes de ejecutar.

3) Composer no estaba disponible.
   - Solución: `full-setup.ps1` descarga `tools/composer.phar`.

4) Error Composer: **OpenSSL extension missing**.
   - Significado: el PHP en uso no tenía `openssl` habilitado; Composer no puede descargar por HTTPS.
   - Solución: habilitar OpenSSL en `php.ini` (Laragon) y reiniciar.

5) Error `Could not open input file: artisan`.
   - Significado: faltaba skeleton completo de Laravel.
   - Solución: `full-setup.ps1` genera Laravel 12 automáticamente si falta `artisan`.

6) Conflicto npm `ERESOLVE` (ApexCharts).
   - Causa: `vue3-apexcharts` requiere `apexcharts >= 5.10.0`.
   - Solución: se actualizó `package.json` para ser compatible.

## Problemas reales de arranque (y fixes)

1) **Faltaba skeleton de Laravel** (no existía `artisan`).
   - Síntoma: `Could not open input file: artisan`
   - Fix: generar skeleton Laravel 12 dentro del contenedor y completar estructura faltante.

2) **504/500 en Nginx**.
   - Causa principal: errores de Laravel por configuración incompleta y luego por assets (Vite).

3) **Vite apuntaba a `0.0.0.0`** en el HTML y dejaba la SPA en blanco.
   - Fix: `vite.config.js` → HMR host `localhost`.

4) **Vue warning (runtime compilation no soportada)** por usar `template` en el root.
   - Fix: `resources/js/app.js` → usar `render: () => h(RouterView)` y limpiar caché Vite.

5) **Vite manifest not found** (`public/build/manifest.json`) en dev.
   - Fix: levantar Vite (contenedor `node`) y asegurar `public/hot`/dev server operativo.

6) **Permisos / cache de Spatie fallaban por permisos de filesystem**.
   - Síntoma: `file_put_contents(...storage/framework/cache/...): Permission denied`
   - Fix:
     - permisos amplios en `storage/` y `bootstrap/cache/` dentro de contenedor
     - `.env` → `CACHE_STORE=array` para desarrollo (evita escrituras en disco)
