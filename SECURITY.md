# Security Policy

## Supported Versions

Este repositorio es un **portfolio**. No hay releases “soportadas” formalmente.

## Reporting a Vulnerability

Si detectas una vulnerabilidad o exposición de datos:

1. **No la publiques en issues**.
2. Contacta por privado con el autor del repositorio y describe:
   - pasos para reproducir,
   - impacto,
   - propuesta de mitigación.

## Datos sensibles

- No se aceptan PRs que incluyan `.env` reales, credenciales, IPs internas, tokens o dumps con datos personales.
