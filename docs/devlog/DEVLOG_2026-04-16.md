## Estado del repo hoy (qué hay implementado)

### 1) Base de datos (migraciones)

Se crearon migraciones para replicar el **schema de referencia** del SQL de producción (`docs/sql/basedatosnueva.sql`) de forma compatible con SQLite y MySQL.

Migraciones creadas en `database/migrations/`:
- `cemn_cementerios`
- `cemn_zonas` (FK a cementerio)
- `cemn_bloques` (FK a zona; `filas`/`columnas` + `tipo`)
- `cemn_sepulturas` (celdas; FK a zona y opcional a bloque; `fila`/`columna`/`estado`)
- `cemn_terceros` (personas)
- `cemn_concesiones` (FK a sepultura + auto-referencia `concesion_previa_id`)
- `cemn_concesion_terceros` (pivot concesión-tercero con `rol`)
- `cemn_difuntos` (FK opcional a tercero y a sepultura)
- `cemn_documentos` (FK a sepultura)
- `cemn_fuentes`
- `cemn_movimientos` (FK a difunto y sepulturas origen/destino)
- `cemn_registro_fuentes` (FK a fuente)

Notas importantes:
- En el SQL real la “unidad” individual es `cemn_sepulturas` (no `cemn_unidades`).
- Se evitó usar tipos exclusivos de SQLite (objetivo: migraciones válidas también para MySQL).

- `CemnConcesionTercero` (pivot como modelo)

## Problemas reales encontrados hoy (y cómo se resolvieron)

1) **PowerShell ExecutionPolicy** bloqueaba scripts (`PSSecurityException`).
   - Solución: ejecutar con:
     - `powershell -ExecutionPolicy Bypass -File .\scripts\...ps1`

2) Los scripts se estaban ejecutando desde `C:\Windows\system32`.
   - Solución: `cd` al directorio del repo antes de ejecutar.

3) Composer no estaba disponible.
   - Solución: `full-setup.ps1` descarga `tools/composer.phar`.

4) Error Composer: **OpenSSL extension missing**.
   - Significado: el PHP en uso no tenía `openssl` habilitado; Composer no puede descargar por HTTPS.
   - Solución: habilitar OpenSSL en `php.ini` (Laragon) y reiniciar.

5) Error `Could not open input file: artisan`.
   - Significado: faltaba skeleton completo de Laravel.
   - Solución: `full-setup.ps1` genera Laravel 12 automáticamente si falta `artisan`.

6) Conflicto npm `ERESOLVE` (ApexCharts).
   - Causa: `vue3-apexcharts` requiere `apexcharts >= 5.10.0`.
   - Solución: se actualizó `package.json` para ser compatible.

# Devlog — 2026-05-05 — Despliegue real en servidor (Portainer) + fixes de 500

Sesión dedicada a **ejecutar el despliegue completo** en el servidor `10.10.20.31` usando el stack de Portainer preparado el 04-05.

## Accesos / puertos usados

- **Web**: `http://10.10.20.31:8182`
- **SSH**: `10.10.20.31:2224` (usuario `conecta`, contenedor `cementerio_php`)
- **MySQL externo (admin)**: `10.10.20.31:3318` (internamente Laravel usa `cementerio_db:3306`)

## Dónde se desplegó el código

Por cómo venía Apache configurado en la imagen `php82-ssh-conecta:1.3`, existía ya un alias:

- `Alias /conecta2 /var/www/html/conecta2/public`

Por tanto, el despliegue se realizó en:

- **Ruta app**: `/var/www/html/conecta2`
- **URL final**: `http://10.10.20.31:8182/conecta2/`

> Nota: esta decisión evita tocar `/etc/apache2` (no hay `sudo` para `conecta`) y aprovecha el alias existente del contenedor.

## Pasos ejecutados (resumen operativo)

1) **Build frontend** en local:
   - `npm run build` → `public/build/*`

2) **Empaquetado y subida**:
   - paquete `.tgz` excluyendo `node_modules/`, `vendor/`, `storage/` y `.env`
   - subida por SCP al contenedor: `/home/conecta/cementerio_deploy.tgz`

3) **Despliegue en el contenedor**:
   - extracción en `/var/www/html/conecta2`
   - `.env` creado desde `.env.server.example`
   - `APP_KEY` generada (`php artisan key:generate --force`)

4) **Dependencias PHP sin composer global**:
   - descarga de `tools/composer.phar` y `php tools/composer.phar install --no-dev --optimize-autoloader`

5) **Storage y permisos**:
   - creación de carpetas necesarias (`storage/framework/{views,cache,sessions}` y `bootstrap/cache`)
   - `php artisan storage:link`
   - permisos amplios en `storage/` y `bootstrap/cache/` (limitación: no hay root)

6) **BD**:
   - `php artisan migrate --force`
   - seeders: `RolesAndPermissionsSeeder` y `AdminUserSeeder`
   - credenciales dev: `admin / admin2026`

## Incidencia real: HTTP 500 al cargar `/conecta2/`

Síntoma:
- `curl http://127.0.0.1/conecta2/` devolvía **500** con cuerpo vacío.

Diagnóstico:
- `storage/logs/laravel.log` mostraba `View path not found` (relacionado con `view:clear`/`optimize:clear`).

Fix:
- asegurar existencia de `storage/framework/views` y limpiar cachés:
  - `php artisan optimize:clear`
- tras el fix, el endpoint respondió **200 OK**.

## Nota Windows: tar + symlink `public/storage`

En Windows, `tar.exe` puede fallar al empaquetar si existe el symlink `public/storage`.
Solución rápida aplicada para poder empaquetar:
- eliminar temporalmente ese symlink en local (se recrea siempre en servidor con `php artisan storage:link`).
