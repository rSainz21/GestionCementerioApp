 Sistema de Gestión y Digitalización de Cementerios Municipales📝 Descripción del ProyectoEste proyecto surge de la necesidad de transformar el modelo analógico y fragmentado de la gestión de los cementerios municipales de Ayuntamiento de Los Corrales de Buelna. El sistema sustituye libros manuscritos antiguos y planos estáticos por una solución digital integral basada en SIG (Sistemas de Información Geográfica).🎯 Objetivos PrincipalesEliminar la Inseguridad Jurídica: Acreditar fehacientemente la titularidad de las concesiones frente a reclamaciones.Regularización de Datos: Ubicar físicamente en el mapa los registros "huérfanos" (difuntos sin ubicación conocida).Control Económico: Gestionar renovaciones de concesiones y detectar impagos para evitar la pérdida de lucro cesante.Expediente Digital 360°: Centralizar documentos, títulos y fotografías de lápidas en una única fuente de verdad.🏗️ Arquitectura TécnicaBackend (Spring Boot)La API está construida bajo un patrón de diseño Multicapa para garantizar escalabilidad y desacoplamiento:Entidades (JPA): Mapeo exacto de las tablas con prefijo cemen_.Repositorios: Consultas optimizadas para la bandeja de regularización y alertas de caducidad.Servicios (Interfaz + Impl): Lógica de negocio para el generador dinámico de bloques y procesos de traslado.Controladores: Endpoints REST para el consumo de la aplicación móvil de campo.Base de Datos (MySQL)Ubicada en el servidor central 10.10.20.31:3317, utiliza el set de caracteres utf8mb4 para soportar anotaciones históricas y coordenadas geográficas de alta precisión.📂 Estructura de MódulosMódulo de Estructura: Generador paramétrico de bloques de nichos, tumbas y panteones.Módulo de Titularidad: Gestión de contratos de concesión, herederos y contactos.Módulo de Operativa: Registro de inhumaciones, exhumaciones y traslados al osario.Módulo Multimedia: Galería fotográfica y repositorio de documentos escaneados por unidad.Módulo Económico: Liquidación de tasas funerarias y padrones de mantenimiento.🛠️ Instalación y ConfiguraciónRequisitos previosJava 17 o superior.IntelliJ IDEA (recomendado).Acceso SSH al servidor central (Puerto 2223).Configuración del EntornoEn el archivo src/main/resources/application.properties, configurar el acceso a la base de datos:Propertiesspring.datasource.url=jdbc:mysql://10.10.20.31:3317/conecta?serverTimezone=UTC
spring.datasource.username=USUARIO
spring.datasource.password=CONTRASEÑA
spring.jpa.hibernate.ddl-auto=validate
📲 Aplicación Móvil (Trabajo de Campo)La aplicación Android es el factor crítico para el éxito del proyecto. Sus funciones principales son:Verificación In Situ: Cotejar datos de la app con la lápida real.Auditoría Visual: Tomar fotos de cada nicho para completar el inventario.Asignación de Huérfanos: Método de "arrastrar y soltar" sobre el mapa para ubicar registros antiguos.🚀 Fases de ImplementaciónCarga Masiva: Procesamiento OCR de listados actuales.Despliegue SIG: Geoposicionamiento de bloques.Campaña de Verificación: Trabajo de campo por empleados municipales.Explotación: Gestión de tasas y atención al ciudadano.
